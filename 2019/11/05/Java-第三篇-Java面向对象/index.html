Java-[第三篇 Java面向对象]

发表于 2019-11-05 | 更新于: 2019-11-19
字数统计: 5.4k 字 | 阅读时长: 22 分钟

面向对象编程和面向过程编程是两种不同的编程思想。

C语言是面向过程语言，Java是面向对象语言。

面向过程的基本单元是函数，面试对象的基本单元是类。

举例来说：制造一辆汽车，面向过程是把每一步的工艺步骤列出，按照顺序执行，最终完成；面向对象是把汽车拆成多个对象，轮胎、发动机、底盘，将多个对象组装，最终完成。

对于复杂问题，宏观上采用面向对象，微观上采用面向过程。

1. 类

类是对事物抽象的描述。类的成员包括5部分：属性，方法，构造器，代码块，内部类。

语法
修饰符 class 类名 {
  属性；
  方法；
  构造器；
  代码块；
  内部类；
}

注意：

  1. class的访问控制符有两种：public和(缺省)。public表示类可以被任何对象访问，(缺省)表示类只能被相同包中的对象访问。一个程序的主类必须是public。
  2. class的修饰符有四种：public，(缺省)，abstract和final。
  3. 类名：首字母大写 + 驼峰原则。
  4. 类的5个成员可以定义0个或多个。

Java的访问控制符有4种：public，protected，(缺省)，private。

1.1 属性

属性(field)，也就是成员变量，表示类的静态特征。

语法
修饰符 类型 属性名 = 值；

注意：

  1. 属性的修饰符有6种：public，protected，(缺省)，private，static，final。
  2. 如果没有对属性进行初始化，Java使用默认值对其进行初始化。

1.2 方法

方法(method)，表示类和对象的行为特征和功能实现。

语法
修饰符 返回值类型 方法名(形参列表) {
  方法体
}

注意：

  1. 方法的修饰符有7种：public，protected，(缺省)，private，static，final，abstract。
  2. 返回值类型可以是基本数据类型，也可以是引用数据类型。
  3. 方法有返回值时，方法体中需要使用return返回该值；方法没有返回值时，返回值类型使用void；方法没有返回值时，也可以使用return结束方法。
  4. 方法名：见名知意，驼峰原则。
  5. 形参列表可以有0个或1个或多个参数，多个参数之间使用”,”，必须声明参数类型。
  6. 方法体：方法功能的具体实现。不调用方法，不会执行方法体。
  7. 在权限修饰符范围内，多个方法之间可以相互调用。

1.2.1 参数传递

Java里方法的参数传递方式只有一种：值传递。即将实参的副本传入方法内，而参数本身不受影响。

  1. 如果传递的是基本数据类型，传递的是具体的值。
  2. 如果传递的是引用数据类型，传递的是地址值。

1.2.2 方法重载

方法重载(overload)：同一个类中，方法名相同，形参列表不同，构成重载。

形参列表不同指：形参类型不同，形参个数不同，形参顺序不同。

注意：

  1. 方法重载，程序会根据形参列表自动进行匹配。
  2. 方法重载与权限修饰符，返回值类型，形参名称无关。

1.2.3 可变形参

可变形参：方法形参的个数可以是可变多个。

语法
修饰符 返回值类型 方法名（形参类型…形参名）{
  方法体
}

注意：

  1. 可变形参里面参数的个数可以是0个或1个或多个。
  2. 可变形参的底层是数组，取值方式和数组一样。
  3. 数组和可变形参类型相同，不构成重载。
  4. 方法中的可变形参只能有一个且必须是最后一个参数。

1.2.4 方法重写

方法重写(override)：子类继承父类以后，可以对父类中的方法进行重写。

如果子类重写了父类中的方法，那么通过子类对象调用父类中被重写的方法，实际上调用的是子类重写的方法。
返回值类型 父类被重写的方法 >= 子类重写的方法
方法名 父类被重写的方法 = 子类重写的方法
形参列表 父类被重写的方法 = 子类重写的方法
权限修饰符 父类被重写的方法 <= 子类重写的方法
抛出异常 父类被重写的方法 >= 子类重写的方法

注意：

  1. 父类被重写的方法的返回值是void，子类重写的方法的返回值必须是void。
  2. 父类方法被private修饰，不可以被重写。

1.3 构造器

构造器(constructor)，也叫构造方法，用于创建对象和给对象进行初始化。

语法
修饰符 类名（形参列表）{
  初始化语句
}

注意：

  1. 构造器的修饰符有4种：public，protected，(缺省)，private。
  2. 构造器必须使用类名作为方法名，且方法名首字母必须大写。
  3. 形参列表可以有0个(空参构造器)或1个或多个参数(有参构造器)，多个参数之间使用”,”，必须声明参数类型。
  4. 构造器没有返回值类型，初始化语句中不能含有return。
  5. 创建对象必调构造器。
  6. 如果类中没有显式声明构造器，系统创建一个默认构造器(空参)；如果类中已经显式声明构造器，系统不再创建默认构造器。
  7. 构造器可以有多个，彼此之间构成重载。

1.4 代码块

代码块(block)，用于将多行代码封装成一个独立的数据体。

语法
修饰符 {
  语句
}

注意：

  1. 代码块的修饰符有2种：(缺省)，static。
  2. 根据是否被static修饰，代码块分为静态代码块(被static修饰)和构造代码块(没有被static修饰)。
  3. 静态代码块在类加载时执行，只执行一次，且优先于构造代码块，用于对静态属性进行初始化，不可以对非静态属性进行初始化，多个静态代码块按照从上到下顺序执行。
  4. 构造代码块在创建对象时执行，每次创建对象都会执行一次，且优先于构造器，用于对不同对象的共性内容进行初始化，多个非静态代码块按照从上到下顺序执行。
  5. 执行顺序：静态代码块 > 构造代码块 > 构造器。

1.5 内部类

内部类(inner class)，表示在类的内部声明另一个类。

语法
修饰符 class 外部类名 {
  修饰符 class 内部类名 {
    类体
  }
}

注意：

  1. 内部类的修饰符有7种：public，protected，(缺省)，private，static，final，abstract。
  2. 内部类名不能与外部类名相同。
  3. 内部类是外部类的类成员，可以访问外部类的其他类成员。
  4. 内部类分为成员内部类(类体内，方法外)，局部内部类(类体内，方法内)和匿名内部类(没有内部类名)。
  5. 根据是否被static修饰，成员内部类分为静态成员内部类(被static修饰)和非静态成员内部类(没有被static修饰)。
  6. 如果要调用局部内部类的方法，必须在局部内部类所在的方法内实例化该类。
  7. 匿名内部类必须在new后面，用于隐含实现类或者接口。

2. 面向对象三大特征

面向对象具有三大特征：封装，继承和多态。

2.1 封装

封装(encapsulation)用来将代码的实现细节隐藏，防止被外部随意更改。

封装性体现在两个方面：

  1. 广义上：使用4种权限修饰符修饰类，属性，方法，构造器，内部类，控制其访问权限。
  2. 狭义上：私有化属性，并提供公共的set/get方法对属性进行赋值和获取(给属性赋值的方法是setXxx，获取属性值的方法是getXxx)。

2.2 继承

继承(inheritance)是从已有的类中派生出新的类，用来减少代码的冗余，提高代码的扩展性，为多态提供前提。

语法
class 子类 extends 父类 {
  类体
}

注意：

  1. Java只支持单继承，不允许多继承(子类只能直接继承一个父类)。
  2. 子类继承父类，可以得到父类的全部属性和方法，但是不可以直接访问父类的私有属性和方法。
  3. 对于父类的私有属性，子类可以通过set/get方法间接访问；对于父类的方法，子类可以进行重写；子类可以定义自己独有的属性和方法。
  4. 子类不继承父类的构造器，子类实例化时所有的构造器默认调用父类的空参构造器(为了方便，定义类时要显式写出空参构造器)。
  5. 父类的概念是相对的，存在直接父类和间接父类。
  6. 所有的类都继承java.lang.Object类。

子类对象实例化过程：

  1. 创建子类对象必调父类构造器。
  2. 子类所有的构造器默认调用父类空参构造器(因为构造器的第一行默认是super())。
  3. 如果父类没有空参构造器，子类构造器必须使用this或super指定调用的构造器。
  4. 在子类构造器中，this和super只能存在一个。

2.3 多态

多态(polymorphism)指同一方法由于调用者不同，具有不同的形态。例如：“吃饭”方法，在中餐厅用筷子吃饭，在西餐厅用刀叉吃饭。

语法
父类 父类引用 = new 子类构造器

注意：

  1. 多态的3个必要条件：继承，方法重写，父类引用指向子类对象。
  2. 父类引用指向子类对象 = 编译时类型和运行时类型不一致。
  3. 父类引用指向子类对象以后，用该父类引用调用子类重写的方法，此时产生多态。
  4. 父类引用不能调用子类独有的方法，必须通过向下转型。
  5. 父类引用指向子类对象 = 向上转型，属于自动类型转换；子类引用指向子类对象的父类 = 向下转型，属于强制类型转换。
  6. 多态是方法的多态，属性没有多态。
  7. 虚拟方法调用，实现动态绑定：编译看左边，运行看右边。

2.4 抽象

抽象(abstract)的作用是定义规范，子类必须提供具体实现。

语法
abstract class 类名 {
  abstract 返回值类型 方法名(形参列表)
}

注意：

  1. 使用abstract修饰的类是抽象类，使用abstract修饰的方法是抽象方法，抽象方法所在的类必须是抽象类。
  2. 抽象类可以包含属性，方法，构造器，代码块，内部类，但是不能被实例化。
  3. 抽象方法只有方法的声明，没有方法的具体实现。
  4. 抽象类只能被子类继承，抽象方法必须被非抽象子类实现。
  5. abstract不可以与private，final和static一起使用。

2.5 接口

接口(interface)的作用是定义更加约束的规范，真正实现规范和具体实现的分离。

语法
访问控制符 interface 接口名 [extends 接口1，接口2] {
  常量；
  抽象方法；
  静态方法；
  默认方法；
}

注意：

  1. 接口的访问控制符有两种：public和(缺省)；接口名的定义规则与类名一致。
  2. 接口和类是两个不同的概念，接口被实现，类被继承。
  3. Java是单继承多实现，接口可以多继承接口，类可以多实现接口。
  4. 接口中可以有常量，抽象方法，静态方法和默认方法(JDK 1.8+)，接口中没有构造器。
  5. 接口中的常量隐式指定为public static final；接口中的抽象方法隐式指定为public abstract。
  6. 接口不能被实例化，只能通过类实现(implements)接口，一个类实现了接口必须实现接口内的所有抽象方法。
  7. 类优先原则：父类和接口有同名方法，通过该类对象调用的是父类的方法。
  8. 接口由于没有抽象类单继承的局限，优先使用。

接口和类之间的多态性：接口引用指向实现类对象。

语法
接口 接口引用 = new 实现类构造器

注意：

  1. 接口引用指向实现类对象以后，用该接口引用调用实现类重写的方法，此时产生多态。
  2. 接口引用不能调用实现类独有的方法，必须通过向下转型。
